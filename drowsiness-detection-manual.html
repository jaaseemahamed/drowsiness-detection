Introduction & Overview

Driver drowsiness is one of the leading causes of road accidents worldwide. According to the National Highway Traffic Safety Administration (NHTSA), drowsy driving causes over 100,000 crashes annually. This manual describes how to take a web-based AI drowsiness detection system and deploy it into a real-world automotive environment.

üéØ Project Summary

This system uses MediaPipe face landmark detection to calculate the Eye Aspect Ratio (EAR) and determine if a driver is falling asleep. When detected, it triggers alerts, notifies emergency contacts, and can interface with the vehicle's control systems.

üîç How EAR Works

The Eye Aspect Ratio (EAR) is a numerical value calculated from the positions of 6 key eye landmarks detected by MediaPipe. When a person's eyes are open, EAR is approximately 0.25‚Äì0.30. When eyes close (sleep or blink), EAR drops significantly. 

        P2    P3
         \    /
    P1 ‚Äî‚Äî+‚Äî‚Äî+‚Äî‚Äî P4
         /    \
        P6    P5

EAR = (|P2-P6| + |P3-P5|) / (2 √ó |P1-P4|)

Threshold: EAR < 0.25 for > 2 seconds ‚Üí Drowsiness Alert!

Auto-Start on Boot

The application must start automatically when the car's device powers on ‚Äî without any manual interaction from the driver.

Method 1: PM2 Process Manager (Recommended for Node.js)

# Install PM2 globally
npm install -g pm2

# Navigate to your project directory
cd /home/pi/drowsiness-app

# Start your app with PM2
pm2 start npm --name "drowsiness-app" -- start

# Save the current process list
pm2 save

# Generate auto-startup script
pm2 startup
# ‚Üí Copy and run the command it outputs

# Verify it's running
pm2 list

Method 2: systemd Service (Linux)

Create the file /etc/systemd/system/drowsiness.service:

[Unit]
Description=Drowsiness Detection App
After=network.target

[Service]
Type=simple
User=pi
WorkingDirectory=/home/pi/drowsiness-app
ExecStart=/usr/bin/npm start
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
# Enable and start the service
sudo systemctl daemon-reload
sudo systemctl enable drowsiness.service
sudo systemctl start drowsiness.service
sudo systemctl status drowsiness.service

Connecting PC/Board to Vehicle

üîå Method 1: OBD-II Port (Safest ‚Äî Start Here)

Every car manufactured after 2000 has an OBD-II (On-Board Diagnostics) port, usually located under the steering column. This is the simplest and safest way to read car data.

Data You Can ReadData You Can Write/Control
‚úÖ Engine RPM‚ö†Ô∏è Speed limiter signals
‚úÖ Vehicle speed‚ö†Ô∏è Engine state
‚úÖ Fuel level‚ö†Ô∏è Gear shift signals
‚úÖ Engine temperatureüî¥ Brakes (requires CAN)
‚úÖ Fault codesüî¥ Steering (requires CAN)
 
# Install python-OBD library
pip install obd

# Python code to read car data
import obd

connection = obd.OBD()  # auto-connects to ELM327

# Read vehicle speed
speed_cmd = obd.commands.SPEED
response = connection.query(speed_cmd)
print(f"Speed: {response.value}")

# Read engine RPM
rpm_cmd = obd.commands.RPM
response = connection.query(rpm_cmd)
print(f"RPM: {response.value}")

üöå Method 2: CAN Bus Interface (Advanced Control)

CAN (Controller Area Network) Bus is the internal communication network of modern vehicles. All sensors, ECUs, and control systems communicate through this bus.

# Install python-can library
pip install python-can

import can

# Connect to CAN Bus via USB interface
bus = can.interface.Bus(channel='can0', bustype='socketcan')

# Listen for messages from the car
for msg in bus:
    print(f"CAN ID: {msg.arbitration_id:#x}, Data: {msg.data.hex()}")

# Send a control command (example: horn)
msg = can.Message(arbitration_id=0x123, data=[0x01, 0x00, 0x00, 0x00])
bus.send(msg)

‚ö° Method 3: Relay Module (Direct Hardware Control)

The simplest method ‚Äî use GPIO pins on your board to control physical relays connected to car components:

# For Raspberry Pi GPIO
import RPi.GPIO as GPIO
import time

# Setup
GPIO.setmode(GPIO.BCM)
GPIO.setup(17, GPIO.OUT)  # Pin 17 connected to buzzer relay
GPIO.setup(18, GPIO.OUT)  # Pin 18 connected to seat vibration relay

def trigger_alert(level):
    if level == 1:
        GPIO.output(17, GPIO.HIGH)  # Activate buzzer
        time.sleep(2)
        GPIO.output(17, GPIO.LOW)
    elif level == 2:
        GPIO.output(18, GPIO.HIGH)  # Activate seat vibration
        time.sleep(5)
        GPIO.output(18, GPIO.LOW)

Alert Escalation System

A proper drowsiness alert system should escalate gradually ‚Äî not immediately triggering emergency responses for a short blink:

LevelTrigger ConditionActions TakenRisk
Level 1
Mild		 EAR < 0.25 for 1‚Äì2 seconds Visual warning on screen + soft beep Low
Level 2
Moderate		 EAR < 0.25 for 2‚Äì5 seconds Loud alarm + seat vibration + voice message Medium
Level 3
Critical		 No response after Level 2, 5+ seconds SMS to emergency contacts + GPS location + reduce engine throttle High

Recommended Implementation Phases

üñ•Ô∏è PC/Laptop in the Car

Mount your existing laptop in the car. Run the web app from the browser. Connect a USB webcam to the dashboard. The software runs as-is ‚Äî no changes needed. This is the fastest way to demonstrate the project working in a real vehicle.

Cost: ‚Çπ0 (use existing laptop) + ‚Çπ500 USB webcam mount

Phase 2 ‚Äî Short Term (Month 1‚Äì2)
üîå OBD-II + SMS Integration

Add an ELM327 OBD-II adapter (‚Çπ800) to read real car speed. When drowsiness is detected AND car is moving, trigger SMS via Twilio/MSG91. Add GPS tracking so emergency contacts receive the exact location. This makes the system fully functional for real-world use.

Cost: ‚Çπ800 OBD adapter + ‚Çπ500/month SMS service

Phase 3 ‚Äî Long Term (Month 3‚Äì6)
üöÄ Embedded System + Vehicle Control

Replace laptop with Orange Pi 5 / Jetson Nano. Add relay module for buzzer and seat vibration. Implement CAN Bus interface for active speed reduction. Deploy to a dedicated enclosure mounted permanently in the vehicle dashboard.
